# Trading Lab — Research Floor (preview)

Example floor for the Office Visual Builder Kit: the `trading-lab` multi-agent
research system rendered as a pixel-art office. Two themes share one floor:
**Day Office** (default) and **Night Control Room** — toggle in the topbar.

## Run

```bash
# from the repo root
npm install
npm run dev          # → http://localhost:5173
```

Controls: drag to pan, wheel to zoom, hover to inspect, click to pin an entity
in the debug card, click empty floor to clear. The topbar has the Day/Night
theme toggle; the "simulate agent activity" checkbox (off by default) cycles
plausible agent statuses (preview-side only — no real API).

## What's on the floor

Agents: Boss/Orchestrator (charcoal suit + gold tie, winged executive chair
behind a deep mahogany command console), Strategy Analyst, Researcher,
Critic/Risk Reviewer, Builder, Evaluator, Performance Monitor. Everyone
faces the viewer, seated behind their desk with a slim aluminum monitor in
front (screen toward the agent, glowing mark on its back); role labels are
nameplate plaques on each desk's front edge. Characters are **composed from
real Universal LPC Spritesheet Character Generator layers** (chair-sit
pose, the generator's own palette ramps) — see
`public/assets/third-party/lpc/` for SOURCE/ATTRIBUTIONS/LICENSES. Agents
sit still — no idle animation.

Layout: one shared plank floor (no zone carpets — desks contrast through
dark espresso wood + aluminum monitors); boards symmetric around the
centered door (Hypothesis left, Backtests right); vending machine + bin
and bin + big water cooler stand flush against the wall flanking the
entrance; the Boss sits screen-center, one row above a glass-walled infra
room (square corners, sliding glass door) holding the Server Rack, Archive
Shelf and Bot Status Monitor; a coffee corner sits by the bottom wall.

Interactive objects: Hypothesis Board, Wall Monitor (backtests), Bot Status
Monitor, Archive Shelf, Server Rack / Data Node, Boss Console (a hit-area
over the console furniture tiles).

## Structure

```text
src/
  main.tsx                                  entry (no StrictMode — see comment)
  TradingLabResearchFloorPreview.tsx        preview shell + theme toggle + simulation
  DebugCard.tsx                             hover/click debug overlay
  scene/tradingLabResearchFloor.scene.ts    semantic scene config (FLOOR_THEMES)
public/
  maps/trading-lab-research-floor-day.tmj   Tiled-compatible maps (generated;
  maps/trading-lab-research-floor-night.tmj  same geometry, per-theme tileset)
  assets/generated/                         original pixel assets (generated)
  assets/third-party/lpc/                   LPC agent sprites + licensing docs
tools/
  generate-assets.mjs                       renders tiles + props (zero deps)
  generate-map.mjs                          builds both .tmj variants
  compose-lpc-agents.mjs                    composes agents from real LPC layers
  lib/                                      palette, tiles, props, png encode/decode
```

## Regenerate assets / map

```bash
npm run generate            # tiles + props + maps
npm run generate:assets
npm run generate:map
ULPC_DIR=~/tmp/ulpc npm run compose:lpc   # agents (needs an LPC generator checkout)
```

Output is deterministic; committed files reproduce byte-for-byte. Edit
`tools/lib/palette.mjs` for colors, `tools/generate-map.mjs` for layout,
`tools/compose-lpc-agents.mjs` (LOOKS) for characters — see
`packages/office-visual-kit/docs/visual-tuning.md`.
